==================================================
Registers
========


Despite being a stack-based VM, the Xenon-131 VM has 5 registers:
* {ANSI.BOLD}FP (Frame Pointer){ANSI.END} - Stores the current frame pointer. Used for subroutines.
* {ANSI.BOLD}SP (Stack Pointer){ANSI.END} - Stores the address that values will get pushed to the data stack.
* {ANSI.BOLD}IP (Instruction Pointer){ANSI.END} - Stores the address of the current instruction.
* {ANSI.BOLD}BP (Base Pointer){ANSI.END} - Stores the base pointer. Used for relative addressing.
* {ANSI.BOLD}IM (Interrupt Mask){ANSI.END} - Stores the interrupt mask.

Before the VM is booted up, the following defaults are loaded:

{ANSI.RED}FP{ANSI.END} <- {ANSI.CYAN}0x00000000{ANSI.END}
{ANSI.RED}SP{ANSI.END} <- {ANSI.CYAN}0x00000000{ANSI.END}
{ANSI.RED}IP{ANSI.END} <- {ANSI.CYAN}0x00000000{ANSI.END}
{ANSI.RED}BP{ANSI.END} <- {ANSI.CYAN}0x00000000{ANSI.END}
{ANSI.RED}IM{ANSI.END} <- {ANSI.CYAN}0xffffffff{ANSI.END}


==================================================
Data Structure Representations
========

{ANSI.BOLD}Integer{ANSI.END}
* Each integer is stored as a 32-bit signed value and occupies 1 memory address.
* Range: `-2147483648` to `2147483647` inclusive.

{ANSI.BOLD}Float{ANSI.END}
* Each float is stored as a 32-bit value (single-precision) and occupies 1 address in memory.
* It adheres to IEEE-754 specifications:

```
x xxxxxxxx xxxxxxxxxxxxxxxxxxxxxxx
_ ________ _______________________
S    E                M

1 sign bit
8 exponent bits
23 mantissa bits

Value = (-1)^S * 2^(E - 127) * (1 + M)
```

* Range: `-3.4e38` to `3.4e38` inclusive.
* Maximum safe integer: `2^24 (16777216)`.

{ANSI.BOLD}Boolean{ANSI.END}
* Booleans are represented using 2 values:
  - {ANSI.PURPLE}false{ANSI.END} = `0x00000000 (0)`
  - {ANSI.PURPLE}true{ANSI.END}  = `0xFFFFFFFF (-1)`
* This allows logical and bitwise AND, OR, NOT, and XOR to share the same opcodes.

{ANSI.BOLD}Pointer{ANSI.END}
* Pointers are represented using a 32-bit integer which value is the pointer's target address.

{ANSI.BOLD}Array{ANSI.END}
* Arrays are dynamically allocated contiguous blocks of memory.
* An array variable stores a pointer to the array's first element.
* Each element occupies one 32-bit memory address, regardless of its type.

{ANSI.BOLD}String{ANSI.END}
* Strings are dynamically allocated contiguous blocks of ANSI characters terminated by a null character (`'\0'`).
* A string variable stores a pointer to a string descriptor.
* The string descriptor contains:
  - Pointer to the character buffer.
  - Current length (including the terminating null character).
  - Allocated capacity.
* Each character occupies one 32-bit memory address, although only the least-significant 8 bits are used.

{ANSI.BOLD}Character{ANSI.END}
* Characters are stored as unsigned 8-bit ANSI values.
* Like every primitive value in the XVM, each character occupies one 32-bit memory address.
* The remaining upper 24 bits are ignored.

{ANSI.BOLD}Subroutine{ANSI.END}
* Subroutines (functions and procedures) are represented by their entry address within program memory.
* Calling a subroutine transfers execution to its entry address while creating a new stack frame.
* A subroutine value therefore occupies one 32-bit memory address.

{ANSI.BOLD}Object{ANSI.END}
* Objects are dynamically allocated blocks of memory.
* An object variable stores a pointer to the first field of the object's memory.
* Object fields are laid out contiguously in declaration order.
* Member access is performed by adding the field's compile-time offset to the object's base address.

Syscalls
========

{ANSI.BOLD}I/O Syscalls{ANSI.END}

{ANSI.CYAN}1 - OUTPUT_CHARS{ANSI.END}
Outputs the characters until a null-terminator is reached from the pointer to the character buffer on top of the stack.

```
Stack before:
`[ &char_buffer ]`

Stack after:
`[]`
```

{ANSI.CYAN}2 - READ_STR{ANSI.END}
Pauses execution in the current process and waits for user input, then creates a new string with the received input.

```
Stack before:
`[]`

Stack after:
`[ &input.descriptor ]`
```

==================================================
I/O Assembly Implementation
========

{ANSI.BOLD}Output String{ANSI.END}

```xe
out << str
```

```xas
LOAD &str  ; &str.descriptor[0]
LOADIND    ; &str[0]
SYS 1      ; output syscall
```

{ANSI.BOLD}Output Character{ANSI.END}

```xe
out << 'A'
```

```xas
PUSH 65  ; 'A'
SYS 9    ; put_char syscall
```

{ANSI.BOLD}Output Integer{ANSI.END}

```xe
out << 42
```

```xas
PUSH 16
SYS 21 ; allocate character buffer
DUP
PUSH 42
SYS 5  ; inttostr syscall
SYS 1  ; output syscall
```

{ANSI.BOLD}Output Float{ANSI.END}

```xe
out << 3.14
```

```xas
PUSH 16
SYS 21           ; allocate character buffer
DUP
PUSH 0x4048F5C3  ; 0x4048F5C3 -> 3.14
SYS 6            ; floattostr syscall
SYS 1            ; output syscall
```

{ANSI.BOLD}Output Boolean{ANSI.END}

```xe
out << true
```

```xas
PUSH 5
SYS 21           ; allocate character buffer
DUP
PUSH 0xFFFFFFFF  ; true
SYS 7            ; booltostr syscall
SYS 1            ; output syscall
```

{ANSI.BOLD}Output Pointer{ANSI.END}

```xe
out << ptr
```

```xas
PUSH 10
SYS 21        ; allocate character buffer
DUP
LOAD &ptr
SYS 8         ; inttohex syscall
SYS 1         ; output syscall
```

The semantic analyzer will handle these cases automatically.
